# Financial Reports Guide

This guide explains the comprehensive financial reporting features available in LandlordHub.

## Reports Overview

The Reports page provides detailed financial analysis and reporting capabilities to help property owners make informed decisions and prepare for tax season.

## Available Reports

### 1. Tax Summary Card
**Purpose**: Quick overview of tax-relevant financial data

**Features**:
- **Total Income**: Sum of all monthly rent income
- **Deductible Expenses**: Total expenses that can be deducted
- **Net Taxable Income**: Income minus deductible expenses
- **PDF Export**: Generate professional tax report for accountant

**Use Cases**:
- Tax preparation and planning
- Quarterly tax estimates
- Year-end tax reporting
- Accountant consultation

### 2. Profit & Loss by Property
**Purpose**: Analyze profitability of individual properties

**Columns**:
- **Property Address**: Property location
- **Monthly Rent**: Monthly rental income
- **Total Expenses**: All expenses for the property
- **Net Income**: Monthly rent minus total expenses
- **ROI %**: Return on investment percentage

**Features**:
- **Sorting**: Properties sorted by net income (highest to lowest)
- **Totals Row**: Summary of all properties
- **Color Coding**: Green for positive, red for negative values
- **Date Filtering**: Filter by specific time periods

**Use Cases**:
- Property performance analysis
- Investment decision making
- Portfolio optimization
- Problem property identification

### 3. Expenses by Category (Pie Chart)
**Purpose**: Visual breakdown of expense categories

**Features**:
- **Interactive Chart**: Hover for detailed information
- **Percentage Display**: Shows both dollar amount and percentage
- **Color Coding**: Distinct colors for each category
- **Responsive Design**: Works on all screen sizes

**Categories**:
- Repairs & Maintenance
- Utilities
- Insurance
- Property Taxes
- Management Fees
- Marketing
- Legal Fees
- Other

**Use Cases**:
- Expense analysis and budgeting
- Tax category planning
- Cost reduction identification
- Budget allocation

### 4. Income vs Expenses Over Time (Line Chart)
**Purpose**: Track financial trends over time

**Features**:
- **12-Month View**: Last 12 months of data
- **Dual Lines**: Income (green) and expenses (red)
- **Trend Analysis**: Visual identification of patterns
- **Intersection Points**: Where income and expenses meet

**Use Cases**:
- Financial trend analysis
- Seasonal pattern identification
- Cash flow planning
- Performance tracking

## Date Range Filtering

### Predefined Ranges
- **This Month**: Current calendar month
- **Last Month**: Previous calendar month
- **This Quarter**: Current quarter (Q1, Q2, Q3, Q4)
- **This Year**: January 1st to December 31st

### Custom Ranges
- **Start Date**: Select any start date
- **End Date**: Select any end date
- **Flexible Periods**: Any time period needed

### Filter Application
- **All Reports**: Date filter applies to all reports simultaneously
- **Real-time Updates**: Reports update immediately when filter changes
- **Consistent Data**: All reports use the same filtered dataset

### Financial Formulas
- **Net Income**: Monthly Rent - Total Expenses
- **ROI %**: (Net Income / Monthly Rent) × 100
- **Taxable Income**: Total Income - Deductible Expenses
- **Category Percentages**: (Category Amount / Total Expenses) × 100

## Troubleshooting

### Common Issues

**Charts not displaying**:
- Check browser JavaScript is enabled
- Verify recharts library is loaded
- Try refreshing the page
- Check for console errors

**Data not updating**:
- Verify date range filter is correct
- Check if data exists for selected period
- Refresh page to reload data
- Verify database connection

**PDF export not working**:
- Check browser popup blockers
- Ensure JavaScript is enabled
- Try different browser
- Check file download permissions

### Performance Optimization

**Large Datasets**:
- Use date range filters to limit data
- Consider breaking reports into smaller periods
- Check browser memory usage
- Close other browser tabs if needed

**Slow Loading**:
- Check internet connection
- Verify database performance
- Consider data archiving for old records
- Contact support if issues persist
